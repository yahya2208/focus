# RR-002: Four-Copy Recovery Rule

**Mandatory after every significant commit.**

## The Four Copies

| # | Method | Purpose |
|---|---|---|
| 1 | Git Tag + Commit | Version history |
| 2 | GitHub Push | Remote backup |
| 3 | Compressed Archive (`.tar.gz`) | Offline snapshot |
| 4 | External Storage (Google Drive / OneDrive) | Disaster recovery |

## When to Execute

- After every phase completion
- After every major feature merge
- Before any deployment operation
- Before any destructive git operation

## Snapshot Naming Convention

```
focus-v2-phase-{X}-snapshot-{YYYY-MM-DD}.tar.gz
```

Example: `focus-v2-phase-minus1-snapshot-2026-07-16.tar.gz`

## Verification

After creating all 4 copies, verify:
- [ ] Git tag exists locally: `git tag -l`
- [ ] GitHub has the push: `git log origin/main`
- [ ] Archive exists and is valid: `tar -tzf <archive>`
- [ ] External storage has the file
