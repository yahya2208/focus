# ERR-001: Engineering Recovery Rule

**Mandatory. Enforced by process. No exceptions.**

## Rule

NEVER perform destructive git operations in the Primary Working Directory:

- ❌ `git checkout --orphan`
- ❌ `git rm -rf .`
- ❌ `git reset --hard`
- ❌ Any command that removes tracked/untracked files en masse

## Required Deployment Methods

- GitHub Actions CI/CD (preferred)
- Git Worktree
- Separate clone/copy

## Before Any Destructive Operation

1. Create a Git Tag
2. Push to GitHub
3. Create a compressed backup

## Before Starting Any New Phase

1. Push all completed work to GitHub
2. Create phase completion tag
3. Verify remote has all files

## Incident Reference

This rule was created after the v1 data loss event on 2026-07-16, where `focus-app/` was permanently destroyed by `git rm -rf .` during a deployment attempt.
